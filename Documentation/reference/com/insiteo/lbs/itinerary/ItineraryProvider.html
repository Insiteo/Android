Class Overview

Class that is the entry point for itinerary requests. You can retrieve an instance of it by calling (ItineraryProvider)LocationProvider.getInstance().getModule(ELocationModule.ITINERARY) Itinerary computation can be done if ITINERARY package is installed. When using dynamicMode (default OFF), itinerary will be refreshed each time location changes.

public ItineraryProvider.ItineraryRequest requestItinerary (ArrayList<Position> departures, ArrayList<Position> destinations, IItineraryRequestListener listener, boolean disabled)

Request shortest route from departures to destinations. Only one departure and one destination will be selected after computation (those defining the shortest path). Selected departure and destination can be retreived by calling getDestination() and getDeparture() on the returned request, after the route is calculated.

Parameters
departures the list of potential departures of the itinerary.
destinations the list of potential destinations of the itinerary
listener the listener that will receive notifications for this request
disabled if true, itinerary will be calculated for persons with limited mobility
Returns
  • the request that will be filled with the itinerary

public void setDynamicMode (boolean isDynamic)

Activate/desactivate dynamic mode. When dynamic mode is ON, itinerary will be refreshed each time location is updated. Dynamic mode must be set to true to received onItineraryChanged(BaseRequest, float) callback.

Parameters
isDynamic true to switch ON dynamic mode, false to switch it of.
